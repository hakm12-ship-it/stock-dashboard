# 주식 분석 대시보드 — 상세 기획서

> **버전** v0.1 · **스택** Streamlit + Python · **형태** 웹 대시보드

한국·미국 종목을 한 화면에서 **기술적 · 기본적 · 뉴스/감성 · AI 리포트** 관점으로 조사하는 **투자 판단 보조** 도구.
매매 결정은 사람이, 근거 수집은 도구가 담당한다.

| 항목 | 내용 |
|---|---|
| **목적** | 투자 판단 보조 (사람이 최종 판단) |
| **시장** | 한국(KOSPI/KOSDAQ) · 미국(NYSE/NASDAQ) |
| **분석** | 기술적 · 기본적 · 뉴스/감성 · AI 요약/리포트 |
| **형태** | 웹 대시보드 (Streamlit) |

---

## 1. 개요

여러 사이트(차트·재무·뉴스)를 오가며 하던 종목 조사를 **하나의 대시보드**로 합친다.
티커 하나를 입력하면 판단에 필요한 근거가 한 화면에 모인다.

### 핵심 원칙
- **수집과 분석의 분리** — 데이터 수집 레이어를 따로 두고 로컬에 캐싱한다. API 호출 한도에 걸리지 않고 화면이 빠르다.
- **MVP는 기술적 분석까지** — 4가지를 한 번에 다 만들지 않는다. 차트+지표만으로도 매일 쓰는 도구가 된다.
- **AI는 마지막에** — 기술·기본·뉴스 데이터가 갖춰진 뒤 Claude가 이를 종합해 리포트를 쓴다. 데이터 없는 요약은 만들지 않는다.

> ⚠️ **고지** — 이 도구는 **투자 조언이 아니다.** 모든 지표·리포트는 참고용 정보이며, 최종 매매 판단과 그 책임은 사용자에게 있다. 화면 곳곳에 이 문구를 노출한다.

## 2. 시스템 아키텍처

3개 레이어. 위에서 아래로 요청이 내려가고, 데이터 레이어의 캐시가 반복 호출을 흡수한다.

```
┌─────────────────────────────────────────────┐
│  프레젠테이션 · Streamlit 대시보드              │
│  종목검색 · 캔들차트 · 지표 · 재무 · 뉴스 · AI리포트 │
└──────────────────┬──────────────────────────┘
                   │ 함수 호출
┌──────────────────┴──────────────────────────┐
│  분석 엔진 · Python 모듈                        │
│  ├ technical.py   이평·RSI·MACD·볼린저          │
│  ├ fundamental.py PER·PBR·ROE·재무             │
│  ├ sentiment.py   뉴스 감성 (Claude)           │
│  └ report.py      AI 종합 리포트 (Claude)       │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────┴──────────────────────────┐
│  데이터 레이어 · 수집 + 캐시                     │
│  provider(KR/US 어댑터) · SQLite 캐시 · TTL 갱신 │
└──────────────────────────────────────────────┘
```

## 3. 데이터 소스

한국·미국을 동시에 다루므로 소스 매핑이 프로젝트의 절반이다.
무료 소스 위주로 시작하고, 실시간이 필요할 때만 유료/증권사 API를 붙인다.

| 데이터 | 한국 (KR) | 미국 (US) | 비용 |
|---|---|---|---|
| 가격 / 시세(지연) | `FinanceDataReader`, `pykrx` | `yfinance`, `FinanceDataReader` | 무료 |
| 실시간 시세 | 한국투자증권 KIS API | Polygon / Finnhub | 무료~유료 |
| 재무 / 공시 | DART Open API | `yfinance`, FMP | 무료 |
| 뉴스 | 네이버 뉴스, RSS | Finnhub, NewsAPI, RSS | 무료~유료 |
| 감성 / 리포트 | Claude API (수집한 뉴스·데이터를 요약·감성분석·종합) | ← | 사용량 |

> 💡 **추천 조합** — `FinanceDataReader` + `DART API`. 한 라이브러리로 KR·US 가격을 모두 커버하고, 재무는 무료 공식 API로 확보한다. 실시간·초단타가 목적이 아니라면 이 조합만으로 Phase 2까지 충분하다.

## 4. 화면 구성

왼쪽 **사이드바**에서 종목을 고르고 시장·기간을 설정한다. 본문은 **탭**으로 분석 관점을 전환한다.
Streamlit의 사이드바 + 탭 구조에 그대로 매핑된다.

```
┌──────────┬──────────────────────────────────┐
│ 사이드바   │  [개요] [기술적] [기본적] [뉴스·AI]  │
│          │                                  │
│ 🔍 검색   │  현재가 71,200  등락 +1.84%        │
│ 시장 KR/US│  RSI 58.3   PER 12.4             │
│ 기간      │  ┌────────────────────────────┐  │
│ 1D·1M·1Y │  │       가격 차트 영역         │  │
│ ⭐관심종목 │  └────────────────────────────┘  │
└──────────┴──────────────────────────────────┘
```

### 탭별 내용
- **개요** — 현재가·등락·핵심 지표 요약 KPI, 미니 차트, 한 줄 AI 코멘트
- **기술적** — 캔들+거래량 차트, 이평선·볼린저 오버레이, RSI·MACD 서브차트
- **기본적** — PER·PBR·ROE·배당, 매출·영업이익 추이, 재무제표 요약표
- **뉴스·AI** — 최신 뉴스 목록 + 감성 태그, Claude 종합 리포트

## 5. 데이터 모델 & 디렉토리 구조

캐시는 처음엔 SQLite 하나로 충분하다. 가격 시계열이 커지면 종목별 Parquet 파일로 옮긴다.

### 캐시 스키마 (SQLite)

| 테이블 | 주요 컬럼 | 용도 |
|---|---|---|
| `symbols` | ticker, name, market, sector | 종목 마스터 / 검색 |
| `prices` | ticker, date, o, h, l, c, volume | 일봉 시계열 (OHLCV) |
| `fundamentals` | ticker, period, per, pbr, roe, eps | 재무·밸류에이션 |
| `news` | ticker, published, title, url, sentiment | 뉴스 + 감성 결과 |
| `reports` | ticker, created, model, body | AI 리포트 캐시 |
| `watchlist` | ticker, added, memo | 관심종목 |

### 디렉토리 구조

```
stock-dashboard/
├─ app.py                  # Streamlit 진입점 (사이드바 + 탭)
├─ data/
│  ├─ provider.py          # KR/US 어댑터 통합 인터페이스
│  ├─ cache.py             # SQLite 읽기/쓰기 + TTL
│  └─ symbols.py           # 종목 마스터 로드/검색
├─ analysis/
│  ├─ technical.py         # 이평·RSI·MACD·볼린저
│  ├─ fundamental.py       # PER·PBR·ROE·재무
│  ├─ sentiment.py         # 뉴스 감성 (Claude)
│  └─ report.py            # AI 종합 리포트 (Claude)
├─ ui/
│  ├─ charts.py            # Plotly 캔들/지표 차트
│  └─ tabs.py              # 탭별 렌더링
├─ .env                    # API 키 (DART, KIS, ANTHROPIC_API_KEY)
└─ requirements.txt
```

## 6. 단계별 로드맵

**Phase 0 → 1이 진짜 MVP.** 여기까지 완성하면 매일 쓸 수 있는 도구가 되고, 이후는 근거를 두껍게 쌓는 작업이다.

### Phase 0 — 데이터 기반 · ~1주
- KR·US **종목 마스터** 구축 + 검색
- `FinanceDataReader`로 가격 수집, SQLite **캐싱** + TTL
- `provider` 인터페이스로 KR/US 차이 흡수
- **산출물** → 티커를 넣으면 OHLCV가 캐시에서 반환되는 데이터 레이어

### ⭐ Phase 1 — 기술적 분석 (MVP) · ~1–2주
- Streamlit 사이드바 **종목 검색** + 기간 선택
- Plotly **캔들 + 거래량** 차트
- **이평선·볼린저** 오버레이, **RSI·MACD** 서브차트
- **산출물** → 종목 검색 → 차트+지표가 뜨는, 그 자체로 쓸모있는 대시보드

### Phase 2 — 기본적 분석 · ~1–2주
- KR `DART API` · US `yfinance`로 재무 수집
- **PER·PBR·ROE·배당** 밸류에이션 카드
- 매출·영업이익 추이 + 재무제표 요약표
- **산출물** → 기본적 탭. 가격과 가치를 나란히 판단

### Phase 3 — 뉴스 + AI 리포트 (Claude) · ~2주
- 종목별 **뉴스 수집** + Claude **감성 태그**
- 기술·기본·뉴스 데이터를 한 프롬프트로 → **종합 리포트 자동 생성**
- 리포트 캐싱으로 API 비용 절감
- **산출물** → "이 종목 지금 상황" 한눈 요약 리포트

### Phase 4 — 종합 & 편의 · 지속
- **관심종목** 저장 + 여러 종목 비교
- 조건 **알림** (지표 돌파, 급등락)
- 리포트 정기 생성 스케줄
- **산출물** → 매일 열어보는 개인 리서치 허브

## 7. AI 리포트 (Claude)

AI는 데이터를 만들어내는 게 아니라 **이미 수집한 데이터를 해석·요약**한다.
수치는 코드가 계산하고, Claude는 그 근거를 사람이 읽을 언어로 종합한다.

```python
# report.py — 계산된 값들을 컨텍스트로 주입
context = {
    "종목": "삼성전자 (005930)",
    "기술적": {rsi, macd_signal, ma_cross, bollinger_pos},
    "기본적": {per, pbr, roe, 매출증가율},
    "뉴스감성": [{title, sentiment}, ...],
}

# Claude Sonnet 5 로 근거 기반 종합 요청
prompt = (
    "위 데이터만 근거로, 강세/약세 요인과 주의점을 "
    "각 3줄로. 투자 조언이 아닌 정보 요약으로."
)
```

- **모델 선택** — 일상 요약·감성분석은 `claude-sonnet-5`(비용/속도 균형), 심층 종합 리포트는 Opus.
- **환각 억제** — 프롬프트에 계산된 수치만 넣고 "제공된 데이터만 근거로" 제약을 건다. Claude가 숫자를 지어내지 않게 한다.
- 정확한 모델 ID·파라미터는 개발 단계에서 `claude-api` 레퍼런스로 확정.

## 8. 리스크 & 주의사항

| 리스크 | 대응 |
|---|---|
| **API 호출 한도** | 무료 소스는 rate limit이 빡빡하다. 캐시 우선, 필요할 때만 원격 호출. TTL로 갱신 주기 관리. |
| **KR/US 데이터 차이** | 티커 형식·통화·거래시간·재무 항목명이 다르다. `provider` 어댑터로 차이를 한 곳에 가둔다. |
| **데이터 정확도** | 무료 소스는 오류·지연이 있다. 실매매 근거 전 최소 2개 소스 교차 확인 고려. |
| **AI 비용·환각** | 리포트를 캐싱하고, 수치는 코드가 계산해 프롬프트에 주입. |
| **법적 고지** | "투자 조언 아님" 문구를 화면·리포트에 상시 노출. 자동매매 연동은 별도 신중 검토. |
| **API 키 관리** | DART·KIS·Anthropic 키는 `.env`로 분리, 저장소에 커밋 금지. |
